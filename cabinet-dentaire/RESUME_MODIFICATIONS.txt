🧪 VÉRIFICATION - AVANT DE TESTER
═════════════════════════════════════════════════════════════════════════════════

Pour vérifier que tout fonctionne:

1. Assurez-vous que la base de données est initialisée:
   → http://localhost/web-hopital/reinit.php

2. Allez à la page login:
   → http://localhost/web-hopital/public/index.php?route=/login

3. Vérifiez visuellement:
   ✅ Page affichée correctement
   ✅ Design moderne avec dégradé
   ✅ Aucun message de debug visible
   ✅ Aucun compte de test affiché
   ✅ Formulaire minimaliste (email + password)
   ✅ Bouton "Se connecter"
   ✅ Responsive sur mobile

4. Testez l'erreur - champs vides:
   - Cliquez "Se connecter" sans rien remplir
   ✅ Doit afficher: "Veuillez remplir tous les champs"
   ✅ Aucun debug ne doit s'afficher
   ✅ Message propre et discret

5. Testez l'erreur - mauvais identifiants:
   - Email: test@test.com / Password: wrongpassword
   ✅ Doit afficher: "Email ou mot de passe incorrect"
   ✅ Aucun debug ne doit s'afficher
   ✅ Pas d'information qui révèle si l'email existe

6. Testez le succès:
   - Email: patient@test.com / Password: 123456
   ✅ Doit afficher: "Connexion réussie ! Bienvenue Jean Dupont"
   ✅ Puis redirection vers: patient/dashboard
   ✅ Aucun debug ne doit s'afficher à aucun moment

7. Vérifiez les autres comptes:
   - medecin@test.com / 123456 → médecin/dashboard
   - secretaire@test.com / 123456 → secretaire/dashboard
