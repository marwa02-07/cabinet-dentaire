AUTHENTICATION FLOW
═══════════════════════════════════════════════════════════════════════════════

1. User visits: index.php?route=/login
   └─ AuthController::login() displays form

2. User enters email & password, clicks Submit
   ├─ POST request → AuthController::authenticate()
   ├─ Validates input (email, password not empty)
   ├─ Calls User::findByEmail($email)
   │  ├─ Prepares query with debug message
   │  ├─ Binds :email parameter
   │  ├─ Executes query
   │  └─ Fetches user (or returns null)
   ├─ Compares password: if ($password === $user['password'])
   ├─ If correct:
   │  ├─ Creates session variables
   │  ├─ Stores debug messages in session
   │  └─ Redirects to dashboard
   └─ If incorrect:
      ├─ Stores error message in session
      └─ Redirects back to login (shows debug messages)

3. Page loads with debug/error messages in yellow alert

ARCHITECTURE - How It All Works
═══════════════════════════════════════════════════════════════════════════════

public/index.php (Front Controller)
  ├─ Defines paths
  ├─ Starts session
  ├─ Initializes autoloader
  ├─ Loads routes from routes/web.php
  ├─ Parses ?route parameter
  └─ Calls appropriate controller method

app/core/Database.php (Connection Manager)
  ├─ Creates PDO connection to MySQL
  ├─ Stores debug messages
  └─ Returns connection via getConnection()

app/core/Model.php (Base Model)
  ├─ Extends Database
  ├─ Collects debug messages
  └─ Provides $pdo to child classes

app/core/Controller.php (Base Controller)
  ├─ Provides view() method
  └─ Renders views with variables

app/models/User.php (User Data Model)
  ├─ Extends Model
  ├─ Query: findByEmail($email)
  ├─ Returns user array or false
  └─ Generates 8+ debug messages

app/controllers/AuthController.php (Authentication)
  ├─ login() - Display form
  ├─ authenticate() - Process login
  └─ logout() - End session

DATABASE STRUCTURE
═══════════════════════════════════════════════════════════════════════════════

users:
  ├─ id (INT, PK, AUTO_INCREMENT)
  ├─ email (VARCHAR 255, UNIQUE)
  ├─ password (VARCHAR 255) - Plain text for development
  ├─ name (VARCHAR 255)
  ├─ role (VARCHAR 50) - 'patient', 'medecin', 'secretaire'
  └─ created_at (TIMESTAMP)

patients:
  ├─ id (INT, PK, AUTO_INCREMENT)
  ├─ user_id (INT, FK to users)
  ├─ telephone (VARCHAR 20)
  └─ address (VARCHAR 255)

medecins:
  ├─ id (INT, PK, AUTO_INCREMENT)
  ├─ user_id (INT, FK to users)
  ├─ specialite (VARCHAR 255)
  └─ telephone (VARCHAR 20)

secretaires:
  ├─ id (INT, PK, AUTO_INCREMENT)
  ├─ user_id (INT, FK to users)
  └─ telephone (VARCHAR 20)

rendez_vous:
  ├─ id (INT, PK, AUTO_INCREMENT)
  ├─ patient_id (INT, FK to patients)
  ├─ medecin_id (INT, FK to medecins)
  ├─ date_consultation (DATETIME)
  ├─ motif (TEXT)
  └─ status (VARCHAR 50) - 'En attente', 'Confirmé', 'Annulé'

consultations:
  ├─ id (INT, PK, AUTO_INCREMENT)
  ├─ patient_id (INT, FK to patients)
  ├─ medecin_id (INT, FK to medecins)
  ├─ date_consultation (DATETIME)
  ├─ diagnostic (TEXT)
  ├─ ordonnance (TEXT)
  └─ notes (TEXT)

TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Problem: "404 - Route non trouvée"
Solution:
  1. Make sure you use: index.php?route=/login (with ?route= parameter)
  2. Check that routes are defined in app/routes/web.php
  3. Verify controller and method exist

Problem: "Connexion à la base de données échouée"
Solution:
  1. Start MySQL in XAMPP Control Panel
  2. Check app/config/config.php for correct DB_HOST, DB_USER, DB_PASSWORD
  3. Use http://localhost/diagnostic.php to test connection

Problem: "Utilisateurs non trouvés" or login fails
Solution:
  1. Run http://localhost/web-hopital/reinit.php
  2. Wait for success message
  3. Go back to login page and try again

Problem: No debug messages showing
Solution:
  1. Check browser's (F12) Console for JavaScript errors
  2. Make sure you're looking for the YELLOW alert box
  3. Try http://localhost/web-hopital/diagnostic.php to check system

Problem: Session not working properly
Solution:
  1. Check that session_start() is at the top of files
  2. Verify cookies are enabled in browser
  3. Check XAMPP PHP session configuration

⚠️ IMPORTANT NOTES
═══════════════════════════════════════════════════════════════════════════════

1. PLAIN TEXT PASSWORDS
   - For development/learning only
   - Passwords are NOT hashed
   - Comparison: if ($password === $user['password'])
   - In production, use bcrypt or similar

2. QUERY PARAMETERS
   - Uses ?route=/path instead of URL rewriting
   - Required for XAMPP/Windows compatibility
   - Handled by index.php router

3. DEBUG MESSAGES
   - All authentication steps are logged
   - Messages shown in yellow alert box
   - Helps troubleshoot login issues
   - Visible only in login.php view

4. SESSION DATA
   - session_start() called in index.php
   - User info stored in $_SESSION
   - Destroyed on logout
   - Debug messages survive redirects
